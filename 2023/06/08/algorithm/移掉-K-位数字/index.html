402. 移掉 K 位数字

难度中等

给你一个以字符串表示的非负整数 num 和一个整数 k ，移除这个数中的 k 位数字，使得剩下的数字最小。请你以字符串形式返回这个最小的数字。

示例 1 ：

输入：num = "1432219", k = 3

输出："1219"

解释：移除掉三个数字 4, 3, 和 2 形成一个新的最小的数字 1219 。

示例 2 ：

输入：num = "10200", k = 1

输出："200"

解释：移掉首位的 1 剩下的数字为 200. 注意输出不能有任何前导零。

示例 3 ：

输入：num = "10", k = 2

输出："0"

解释：从原数字移除所有的数字，剩余为空就是 0 。

提示：

  • 1 <= k <= num.length <= 105
  • num 仅由若干位数字（0 - 9）组成
  • 除了 0 本身之外， num 不含任何前导零

# 贪心 + 单调栈

若只删除 111 次。

例如： 143221914322191432219，那么肯定是删除 444

  • 若删除 444 后面的元素 322193221932219 其中一个，那么最后肯定是前缀为 14⋯14\cdots14⋯ 的序列

    若删除 444，那么前缀变成 13⋯13\cdots13⋯ 的序列

    二者的长度必然相等，因为都是删除 111 次

    贪心地选择删除 444

例如：145221914522191452219，那么肯定删除 555

  • 对于 555 后面的元素与上述同理

  • 若删除 555 前面的元素 141414 其中一个，那么最后 15⋯15\cdots15⋯ 或者 45⋯45\cdots45⋯ 的序列必然大于 14⋯14\cdots14⋯ 的序列

    贪心地选择删除 555

  • 删除 555 后的序列 142219142219142219，若还可以删除继续删除，那么必然删除 444，分析同上。

若是连续递增的序列，那么不必删除

若当前元素 s[i]s[i]s[i] 小于结尾的元素，那么不断地删除结尾的元素直至 s[i]s[i]s[i] 大于结尾元素，最后将 s[i]s[i]s[i] 加入结果集的尾部

  • 例如：145145145，s[i]=2s[i] =2s[i]=2，那么需要不断地删除结尾元素 145⋯145\cdots145⋯ 至 142⋯142\cdots142⋯ 至 12⋯12\cdots12⋯

最多只能删除 kkk 次，若 k==0k == 0k==0 那么可以直接退出循环，将 sss 中剩余地元素放入结果集

对于 143221914322191432219，k=2k = 2k=2

根据上述步骤删除了 4,34,34,3 此时结果集 121212 ，最后将 219219219 加入结果集。

为什么不删除 219219219 其中一个？

上文已经分析了，对于 333：

  • 若删除 333 以后的元素 221922192219 其中一个，那么最后肯定是 143⋯143\cdots143⋯ 的序列

  • 若删除 333，那么最后是 142⋯142\cdots142⋯ 的序列

    此时后者必然小于前者，贪心的选择后者，即：删除 333

444 也同理

若 kkk 最后还有剩余，那么可以直接删除尾部的 kkk 个元素，若结果集不足 kkk 个，那么直接返回 000

我们可以用栈来表示结果集，可以发现这是个单调递增栈。

  • 即：栈顶元素 peekpeekpeek 大于 s[i]s[i]s[i]，那么不断地删除栈顶元素并更新 k=k−1k=k-1k=k−1 直至栈顶元素 peek<=s[i]peek <= s[i]peek<=s[i]。将 s[i]s[i]s[i] 压入栈顶。
