1015. 可被 K 整除的最小整数

难度中等

给定正整数 k ，你需要找出可以被 k 整除的、仅包含数字 **1** 的最 小 正整数 n 的长度。

返回 n 的长度。如果不存在这样的 n ，就返回 - 1。

注意： n 不符合 64 位带符号整数。

示例 1：

输入：k = 1

输出：1

解释：最小的答案是 n = 1，其长度为 1。

示例 2：

输入：k = 2

输出：-1

解释：不存在可被 2 整除的正整数 n 。

示例 3：

输入：k = 3

输出：3

解释：最小的答案是 n = 111，其长度为 3。

提示：

  • 1 <= k <= 10^5

# 求解 (欧拉定理 + 欧拉函数求解 + 快速幂)

快速幂算法：

如需求数 a 的幂次，此处 a 可以为数，常规做法需要对 a 进行不断的乘积即 a * a * a * ... 此处的时间复杂度将为 O (n)

以求 3103^{10}310 的结果为例：

[优化步骤 1：]

易知：

310=3∗3∗3∗3∗3∗3∗3∗3∗3∗33^{10}=3*3*3*3*3*3*3*3*3*3310=3∗3∗3∗3∗3∗3∗3∗3∗3∗3

=95=94∗9=9^5 = 9^4*9=95=94∗9

=812∗9=81^2*9=812∗9

=6561∗9=6561*9=6561∗9

基于以上原理，我们在计算一个数的多次幂时，可以先判断其幂次的奇偶性，然后：

  • 如果幂次为偶直接 base (底数) 作平方，power (幂次) 除以 2

  • 如果幂次为奇则底数平方，幂次整除于 2 然后再多乘一次底数

[优化步骤 2：]

对于以上涉及到 [判断奇偶性] 和 [除以 2] 这样的操作。使用系统的位运算比普通运算的效率是高的，因此可以进一步优化：

  • 把 powerpower % 2 == 1power 变为 (power & 1)==1(power\, \&\, 1) == 1(power&1)==1

  • 把 power=power/2power = power / 2power=power/2 变为 power=power>>1power = power >> 1power=power>>1
